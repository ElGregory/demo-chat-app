## Key Conventions

**Route + server handler colocation** вЂ” API routes live alongside their page routes. A `.ts` file at `routes/demo/api.ai.chat.ts` is the server handler; `routes/demo/ai-chat.tsx` is the page.

**AI tool split** вЂ” tools are defined once with `toolDefinition()`, then split:

- `.server(handler)` вЂ” runs on the server, passed to `chat()`
- `.client(handler)` вЂ” runs in the browser, passed to `clientTools()`

**Chat hook pattern** вЂ” create a named hook that wraps `useChat(createChatClientOptions(...))` and export it from `lib/`. Components consume the hook, not raw AI primitives.

**Multi-provider pattern** вЂ” server route checks env vars at runtime to pick a provider; falls back to Ollama. Never hardcode a single provider.

**New feature files** вЂ” place under:

- `src/components/chat/` вЂ” chat UI components
- `src/components/profile/` вЂ” profile display components
- `src/lib/ai/` вЂ” AI orchestration logic
- `src/lib/tools/` вЂ” tool definitions
- `src/lib/profile/` вЂ” profile extraction and conflict detection
- `src/types/` вЂ” shared TypeScript types and Zod schemas

## Do Not Modify

- `src/routeTree.gen.ts` вЂ” auto-generated by TanStack Router
- `src/styles.css` вЂ” global styles
- Files under `src/integrations/` вЂ” unless explicitly extending them
